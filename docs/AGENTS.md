# AGENTS.md

## 対象範囲
- このファイルは `docs/` 配下に適用されます。

## 最初に確認すること
- `docs/README.md` を確認し、既存の文書構成と配置方針を把握してください。
- 特定領域に関する文書を扱う場合は、その領域の `README.md` も確認し、入口情報との役割分担を崩さないでください。

## 役割分担
- `README.md`: 入口
- `docs/`: 詳細説明と横断的なドキュメント
- `AGENTS.md`: 作業ルール
- `docs/adr/`: 意思決定の理由や背景

## 配置ルール
- システム全体の説明は `docs/architecture/` に置いてください。
- 開発手順、セットアップ、テスト方針は `docs/development/` に置いてください。
- 領域別の詳細は `docs/backend/`、`docs/frontend/`、`docs/infra/` に置いてください。
- 重要な設計判断や運用判断は `docs/adr/` に置いてください。
- 入口説明だけで十分な内容は、無理に `docs/` に増やさず、最寄りの `README.md` に書いてください。

## 記述ルール
- 新規文書を作る前に、既存文書の更新で足りないか確認してください。
- 結論、背景、詳細の順で書いてください。
- 推測を事実のように書かないでください。不明なことは不明と明記してください。
- 実在するコマンド、パス、設定だけを書いてください。
- 見出しを使って読みやすく整理してください。
- 新しい詳細文書を追加した場合は、`docs/README.md` または関連する `README.md` から辿れるようにしてください。
- ドキュメントは 必ず日本語で生成・更新してください。
- 構成図があるとわかりやすい場合は、文章よりも構成図（Mermaid）を記載して文章を補足してください。

## 更新ルール
- コード変更に伴う文書更新では、まず最寄りの `README.md` を更新すべきか確認してください。
- 詳細説明や横断的な説明が必要な場合に `docs/` を更新してください。
- 既存文書を更新した場合は、関連リンクや参照先も確認してください。

## 完了条件
- 文書の置き場所が適切であること。
- 既存文書と不必要に重複していないこと。
- 必要に応じて `docs/README.md` や関連 `README.md` から辿れること。
- 記載内容がコードや設定と矛盾していないこと。

## 禁止事項
- 未確認の仕様や運用を断定しないでください。
- 孤立した文書を増やさないでください。
- 入口文書に書くべき内容まで過剰に `docs/` に寄せないでください。
