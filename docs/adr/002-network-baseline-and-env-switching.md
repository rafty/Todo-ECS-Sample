# ADR 002: ネットワーク基盤と環境切替方式の決定

## ステータス
採用

## コンテキスト
- Todo アプリのインフラを CDK で再現可能に管理するため、デフォルト VPC 依存を排除した基盤が必要だった。
- `dev` / `stg` / `prod` の環境差分（アカウント・リージョン）を、コード改変なしで切り替える運用が必要だった。

## 決定事項
- 新規 VPC を作成し、2AZ / 3層サブネット（`front` / `application` / `datastore`）を採用する。
- CDK 実行時は `-c env=<dev|stg|prod>` を必須とし、環境設定は `infra/lib/config/environment-config.ts` から読み込む。
- 共通タグとして `env` / `service` / `version` を付与し、`version` は `1.00` とする。
- Security Group の具体ルールは本 ADR の対象外とし、後続の ALB/ECS/DB 実装時に決定する。

## 根拠
- 環境指定必須化により、誤アカウント・誤リージョンへのデプロイリスクを低減できる。
- 3層分離により、将来の ALB/ECS/Aurora 配置責務を明確化できる。
- タグ統一により、運用時の識別性と変更追跡性を確保できる。

## 影響
- CDK 実行時に必ず `env` 指定が必要になる。
- 実行環境の認証状態（AssumeRole / 証明書チェーン）により `synth` が失敗する場合があり、実行前の認証整備が必要になる。
- SG 詳細は後続実装に持ち越すため、関連機能実装時に追加 ADR または設計記録が必要になる可能性がある。

## 備考
