# AGENTS.md

## 対象範囲
- このファイルはリポジトリ全体に適用されます。
- より深いディレクトリにある `AGENTS.md` が存在する場合は、そちらも確認し、このファイルとあわせて適用してください。
- ルートの `AGENTS.md` は全体ルールを定義し、サブプロジェクト配下の `AGENTS.md` は各領域固有のルールを定義します。

## まず確認すること
- リポジトリ全体の概要、主要ディレクトリ、共通の起動方法や参照先はルートの `README.md` を確認してください。
- 詳細な仕様、設計、手順は `docs/` を確認してください。
- 作業対象の領域に入ったら、そのディレクトリの `README.md` と `AGENTS.md` を確認してください。

## サブプロジェクト別 AGENTS.md の適用ルール
- `backend/` 配下のコードを書いたり修正したりする場合は、ルートの `AGENTS.md` に加えて `backend/AGENTS.md` も確認し、両方のガイドラインを適用してください。
- `frontend/` 配下のコードを書いたり修正したりする場合は、ルートの `AGENTS.md` に加えて `frontend/AGENTS.md` も確認し、両方のガイドラインを適用してください。
- `infra/` 配下のコードを書いたり修正したりする場合は、ルートの `AGENTS.md` に加えて `infra/AGENTS.md` も確認し、両方のガイドラインを適用してください。
- `docs/` 配下のコードを書いたり修正したりする場合は、ルートの `AGENTS.md` に加えて `docs/AGENTS.md` も確認し、両方のガイドラインを適用してください。
- 複数のサブプロジェクトにまたがる変更を行う場合は、変更対象となるすべてのサブプロジェクトの `AGENTS.md` を確認し、関連するガイドラインをすべて適用してください。
- サブプロジェクト配下の `AGENTS.md` にルート `AGENTS.md` より詳細な指示がある場合は、その領域ではサブプロジェクト配下の指示を優先してください。
- サブプロジェクト配下の `AGENTS.md` を自動では参照しない環境でも、対象ディレクトリに応じて明示的にその `AGENTS.md` を確認し、指示に従ってください。

## リポジトリ概要
- このリポジトリには以下が含まれます:
    - `backend/`: Spring Boot を用いたバックエンドアプリケーション
    - `frontend/`: React を用いたフロントエンドアプリケーション
    - `infra/`: AWS CDK (TypeScript) によるインフラ定義
    - `docs/`: リポジトリ全体および各領域のドキュメント
    - `docs/adr/`: 重要な設計判断や運用判断の記録

## 全体ルール
- 変更は依頼された範囲に限定してください。
- 既存のディレクトリ構成、命名規則、実装方針を尊重してください。
- 明示的な依頼がない限り、大規模な横断リファクタリング、広範囲な rename、ファイル移動は避けてください。
- 複数領域にまたがる変更は、本当に必要な場合だけにしてください。
- 自動生成物は、このリポジトリでソースとして管理する運用でない限り直接編集しないでください。

## ドキュメント更新ルール
- 公開挙動、API 契約、環境変数、デプロイ手順、運用手順、監視設定を変更した場合は、対応する `README.md` と `docs/` の更新要否を確認してください。
- 重要な設計判断や運用判断の追加・変更がある場合は、`docs/adr/` への記録を検討してください。
- ドキュメントは必ず日本語で生成・更新してください。
- 構成図があるとわかりやすい場合は、文章よりも構成図（Mermaid）を記載して文章を補足してください。

## 検証
- 変更した領域に対応する検証のみ実行してください。
- 複数領域を変更した場合は、変更した各領域を検証してください。
- 既存のスクリプト、wrapper、CI に合わせたコマンドを優先してください。
- 検証を実行できない場合は、未確認の内容と理由を明記してください。

## 完了条件
- 変更が依頼範囲に収まっていること。
- 関連する検証が完了していること、または未実施理由が明確であること。
- 必要な `README.md` と `docs/` の更新要否を確認していること。
- シークレット、認証情報、state ファイル、不要な差分が含まれていないこと。

## 禁止事項
- シークレット、`.env` の実値、秘密鍵、認証情報、state ファイルをコミットしないでください。
- 本番影響のある設定を、明示的な依頼なしに変更しないでください。
- 無関係な整形だけの大規模差分を作らないでください。
